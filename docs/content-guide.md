# 网站内容规范

本规范用于保证不同批次文章在正文、翻译、粗体、文化拓展、音频和图片上保持一致。正式发布前应逐项检查，不以网页“能够打开”作为唯一完成标准。

## 1. 文章基本字段

每篇文章至少包含：

- `id`：稳定且唯一的文章标识。
- `title`：法语标题，保留正确的重音符号和大小写。
- `chineseTitle`：简洁、自然的中文标题。
- `level`：`A2`、`B1` 或 `B2`。
- `topic`：书中主题标识，如 `food`、`people`、`history`。
- `summary`：首页目录使用的一句中文简介。
- `groups`：按原文自然段组织的法语句子。
- `sentenceTranslations`：与正文逐句对应的中文。
- `highlights`：正文需要加粗的日常表达和固定搭配。
- `culture`：三段法语文化拓展及对应中文。
- `audioPack`：音频包路径和每句的开始、结束时间。

已使用的主题标识：

| 标识 | 法语 | 中文 |
| --- | --- | --- |
| `food` | Alimentation, gastronomie | 食物、美味 |
| `people` | Célébrités | 名人 |
| `history` | Histoire, institutions | 历史、制度 |
| `work` | Industrie, éducation, travail | 工业、教育、工作 |
| `culture` | Langue, médias, culture | 语言、媒体、文化 |
| `daily` | Vie quotidienne | 日常生活 |

## 2. 文件与标识命名

- `id` 使用小写拉丁字母和连字符，不使用空格、法语重音符号或中文。
- 多词标题使用连字符，例如 `le-petit-dejeuner`。
- 音频包：`audio-packs/<article-id>/01.bin`。
- 页面图片：`images/<article-id>-hero.webp`。
- `id` 一经发布尽量不要修改，因为它同时决定文章网址、音频目录和图片文件名。

示例：

```text
文章网址：/french-reading-site/article/le-petit-dejeuner
音频文件：audio-packs/le-petit-dejeuner/01.bin
图片文件：images/le-petit-dejeuner-hero.webp
```

## 3. 级别标注

- 级别限定为 A2、B1、B2，与首页筛选选项保持一致。
- 判断时综合句子长度、词汇频率、语法复杂度、隐含文化知识和篇章推理要求。
- 同一批文章应使用一致尺度；不因主题看似熟悉就自动降低等级。
- 级别是学习导航，不等同于正式考试认证。存在疑问时记录复核意见，再统一调整。

## 4. 法语正文

- 忠实校对重音符号、撇号、连字符、引号、标点和专有名词。
- 保留原文自然段；程序中的 `groups` 应反映段落结构。
- 每个数组元素只放一个可独立点读的完整句子。
- 不为了缩短音频随意拆开语义完整的句子；过长句可在自然停顿处谨慎拆分，并同步翻译和音频。
- 教材正文与本站补写内容必须在来源记录中区分。

## 5. 逐句中文翻译

- `sentenceTranslations` 的数量和顺序必须与 `groups.flat()` 完全一致。
- 每条中文只对应同序号的一条法语句子，不跨句合并，也不把一句拆成多条显示。
- 翻译优先保证含义准确、句序可对照，同时保持自然中文。
- 人名、机构名、历史概念等采用通行译名；不确定时先查证再发布。
- 双击法语句子后直接显示译文，不额外添加“中文”提示。
- 修改法语句子后必须重新核对对应中文和音频，不能只改其中一项。

## 6. 粗体表达

粗体用于帮助学习者发现可迁移的语言表达，不用于装饰。

优先选择：

- 日常口语，如 `prendre l’apéro`。
- 固定搭配，如 `avoir lieu`、`à l’occasion de`。
- 常见句型或连接表达，如 `c’est pourquoi`、`quelles que soient`。
- 在其他主题中仍有复用价值的动词短语、介词短语和副词表达。

一般不选择：

- 单纯的专有名词、人名、地名和品牌名。
- 只在本篇中出现、缺少迁移价值的冷僻名词。
- 过长的整句或一段中密集重复的普通词。

对应规则：

1. 法语表达录入 `highlights`，必须能在正文中准确匹配。
2. 中文译文中对应的意义使用 `**粗体**` 标记。
3. 法语和中文的粗体范围应在意义上对应，不要求机械逐词相等。
4. 若法语已加粗而中文没有对应粗体，或反之，视为发布错误。

## 7. 文化拓展

- 每篇固定三段，每段围绕不同角度补充背景，不重复正文。
- 每段法语约 100–200 词；语言难度应与文章级别大体相称。
- 内容可涉及历史沿革、社会习惯、地区差异、制度背景或当代争议。
- 避免把法国社会写成单一、静止的文化；涉及地域、阶层、性别、族群或历史争议时说明差异和变化。
- 事实性内容应核验；所用外部资料录入 `sources.md`。
- 每段法语后紧跟一个中文自然段；中文按法语句序对应，尽量做到一句对一句。
- 数据结构同时保留完整中文 `zh` 和逐句数组 `zhSentences` 时，两者内容应一致。

## 8. 音频

- 使用自然、清晰的法语女声，当前选用的声音标识为 Vivienne。
- 默认播放速度为自然语速，并提供 0.85× 慢速；不通过生成僵硬的“逐词慢读”替代正常语流。
- 发音、联诵、重音和专有名词读法应抽听检查，避免机械感、截断、爆音和长时间空白。
- 当前每篇文章使用一个 `01.bin` 音频包；文件内部为单声道 MP3 数据，网页按时间段点读。
- `audioPack.segments` 的数量必须等于正文句子数；每段的 `start` 小于 `end`，相邻时间段不得错误重叠或遗漏。
- 替换音频后必须同时更新分句时间，并实际测试全文连续播放、任意句点读、暂停和切换语速。
- 语音服务、声音名称、生成日期和使用许可应记录在 `sources.md`。

## 9. 图片

- 当前命名为 `<article-id>-hero.webp`，使用 WebP 格式。
- 建议宽度约 1600 像素，横向构图；主体不要妨碍标题、按钮和正文辨识。
- 图片只作辅助，不承载理解文章所必需的信息；即使加载失败，文章仍应完整可读。
- 控制文件体积，避免明显降低首页和文章页加载速度。
- 优先使用自行创作、明确授权、公共领域或许可清楚的素材。
- 保存原始来源网址、作者、许可、获取日期和必要的署名要求；信息不全时不得写成“可自由使用”。

## 10. 生成与部署

- `index.html` 是静态成品，包含样式、文章数据和交互代码。
- `404.html` 应与 `index.html` 完全一致，以支持文章直达网址。
- 需要被搜索引擎单独收录的文章应生成 `article/<article-id>/index.html`，返回真实静态页面并设置独立标题、摘要和规范网址。
- GitHub Pages 项目基础路径固定为 `/french-reading-site`。
- 新文章需要同时存在数据、翻译、音频包；美食类页面还应检查对应图片。
- 正式更新优先修改生成源数据并重新生成成品，不在两个大型 HTML 文件中分别手改相同内容。

## 11. 发布前检查清单

- [ ] 标题、级别、主题和首页简介正确。
- [ ] 法语重音符号、标点、分段和句子顺序已校对。
- [ ] 法语句子数等于逐句中文数。
- [ ] 所有法语粗体在中文中都有意义对应的粗体。
- [ ] 粗体以日常表达和固定搭配为主，不把专有名词当学习重点。
- [ ] 三段文化拓展均有按句序对应的中文。
- [ ] 文化事实和引用资料已核验并登记来源。
- [ ] 音频句数、分段时间和正文句数一致。
- [ ] 全文朗读、逐句点读、暂停、0.85× 和 1× 均正常。
- [ ] 单击句子播放，双击句子显示或收起中文。
- [ ] 图片清晰、体积适当、不影响文字可读性，来源记录完整。
- [ ] 首页按级别和按主题筛选正常，文章计数正确。
- [ ] 首页、文章直达链接和不存在的路径均可加载网站。
- [ ] 独立静态文章页的标题、摘要、规范网址和站点地图记录正确。
- [ ] `index.html` 与 `404.html` 内容一致。
- [ ] `CHANGELOG.md` 和 `sources.md` 已随发布更新。
